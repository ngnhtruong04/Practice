12\. Temperature là tham số điều khiển mức độ ngẫu nhiên (Randomness) trong quá trình mô hình lựa chọn token tiếp theo 

Đơn giản hơn: 

Temperature thấp --> Mô hình ưu tiên chọn token có xác suất cao nhất

Temperature cao --> mô hình sẵn sàng chọn các token có xác suất thấp hơn

Temperature không làm mô hình thông minh hơn, mà chỉ thay đổi cách mô hình lấy mẫu từ phân phối xác suất







13\. Top\_p là tham số điều khiển Nucleus Sampling

khác với temperature, Top\_p không làm thay đổi xác suất

Top\_p chỉ quyết định: "Những token nào được phép tham gia bốc thăm"





Cộng đồn xác suất cho tới lớn hơn hoặc bằng Top\_p









Tempareture						Top\_p

\---------------------------------------------------------------------------

Thay đổi phân phối xác suất			Không thay đổi xác suất

Làm phẳng hoặc sắc phân phối			Cắt bớt tập token

Điều chỉnh mức ngẫu nhiên			Điều chỉnh phạm vi lấy mẫu

Ảnh hưởng mọi token				Chỉ giữ lại một phần token

\- Có vượt rate limit không  ?

15\. Các lỗi phổ biến khi gọi API



|Loại lỗi |Nguyên Nhân |Xử lý|
|-|-|-|
|401 - Unauthorized|API Key sai (vd: Invalid API Key)|Kiểm tra:<br />GROQ\_API\_KEY = \_\_\_|
|403 - Forbidden|Không có quyền truy cập :<br />- API bị khóa <br />- Account bị giới hạn ||
|404 - Not Found|Sai endpoint<br />(vd: https//api.groq.com/**wrong**)||
|429 - Rate Limit|Gửi quá nhiều request (Too many request)|- Retry<br />- Back off<br />- Queue request|
|500 - Server Error|Lỗi phía server|Retry sau vài giây |
|Timeout|Request quá lâu |timeout = 30|
|Connection Error|- Mất mạng<br />- DNS lỗi<br />- Server không phản hồi||
